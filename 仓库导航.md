## 1. 总览：逻辑分工

课题采用 **设计基线 + 实现子树** 分离（ADR-0002 的职责分工）。设计定义「应该做什么」，实现证明「已经跑通」。
当前本地 checkout 是单 Git 工作区；`bytedance-implementation/` 不是独立 `.git` 仓。最终对外提交时，应将 **实现子树** 发布为公开 AI 系统主仓，且公开内容包含 `sandbox-repo/` Conduit 代码目录；不是直接向上游 Conduit 仓库提交 PR。

```text
/Users/xioshark/Desktop/bytedance/          ← 工作区根层（设计基线）
└── bytedance-implementation/               ← 实现子树（全部代码与证据）
```

| 区域 | 绝对路径 | 放什么 | 不放什么 |
|------|----------|--------|----------|
| **根层设计基线** | `/Users/xioshark/Desktop/bytedance` | PRD、规格、计划、验收 YAML、进度摘要、源题 PDF | 业务源码、Conduit fork、run 证据、真实 API key |
| **实现子树** | `…/bytedance-implementation` | Web/API/Orchestrator/Agent/Skill、sandbox-repo、run 归档、提交材料 | 重复的设计规格副本、真实密钥入库 |

**Agent / 协作者规则**：代码类工作 **只能在实现子树** 进行；改需求/规格/计划 **只在根层 `docs/`** 对应文件。详见 [`AGENTS.md`](./AGENTS.md)。


### 3.1 核心代码目录职责

| 目录 | 职责 |
|------|------|
| `apps/web` | PM 输入、阶段进度、ClarificationsPanel、AI Usage（单 run + 跨 run）、断点重放入口；`App.jsx` 只装配控制台，`useConsoleController.js` 承载 Web 状态与 action 胶水 |
| `apps/api` | REST + SSE；`app.js` 只做 Express 装配；`runRoutes.js` 只聚合 run 路由，`runExecutionRoutes.js` / `runEvidenceRoutes.js` / `runReviewRoutes.js` 分别承载执行、证据读取、人工确认/PR/submission |
| `services/orchestrator` | clarify → plan → edit → verify → pr 状态机；写 `docs/reports/runs/` |
| `services/agents` | 各阶段 Agent；Requirement 支持 `AI_MODE=rules` 与 `llm` |
| `services/skills` | Skill 注册与匹配；当前 4 个 Skill（见下表） |
| `services/sandbox` | 对 `sandbox-repo` 读文件、打 patch、跑 lint/test |
| `external/git-provider` | 有 `GITHUB_TOKEN` 时创建 draft PR |
| `external/model-client` | LLM API（OpenAI 兼容）；写入 `ai-calls.jsonl` |
| `sandbox-repo` | **真实** Conduit 代码；所有 `diff.patch` 须含此路径 |

### 3.3 Run 证据目录（`docs/reports/runs/<run-id>/`）

每次 DeliveryRun 归档一组文件，用于审计与答辩举证：

| 文件 | 含义 |
|------|------|
| `requirement.md` | 结构化需求卡片；LLM run 含 `clarifications[]` |
| `history-recall.json` | 相似历史 run 召回结果 |
| `plan.md` | 方案与 Skill 定位；可含 `history_references` |
| `diff.patch` | 对 `sandbox-repo` 的真实 git diff |
| `verification.json` | lint / 单测结果 |
| `ai-calls.jsonl` | 模型调用留痕（tokens、latency、model） |
| `pr-draft.md` | PR 标题、描述、风险 |
| `checkpoints.json` | 断点重放 checkpoint |
| `run-summary.json` / `metadata.json` | 运行摘要与元数据 |
