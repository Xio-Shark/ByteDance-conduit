# ADR-0002: 文档仓库与实现仓库分离

## 状态

已接受 (Accepted) — 2026-05-20

## 背景

当前仓库用于重构课题文档和设计目标。若同时放入 Conduit fork、前端、后端和编排源码，容易让文档基线与一次性实现混在一起，增加评审和维护成本。

课题目标仍然是实现真实系统：前端、Node 后端、AI 编排层、Conduit sandbox、验证命令和 PR 材料都必须在实现仓库中落地。

## 决策

本仓库只承载课题设计、架构文档、计划以及验收与追踪基线；业务实现放在独立实现仓库中完成。

## 边界

| 仓库 | 包含 | 不包含 |
|------|------|--------|
| 本文档仓库 | 设计文档、计划、验收与追踪基线 | 业务源码、Conduit fork、真实密钥、Agent spec、模板、Playbook、**根目录 CSV 执行台账** |
| 实现仓库 | 前端、后端、编排、Skill、sandbox-repo、测试、运行报告、提交材料 | 过时文档、真实密钥提交 |

## 影响

- 本仓库可以稳定表达“应该实现什么”。
- 实现仓库负责证明“已经真实跑通”。
- 评审材料需同时引用本仓库文档和实现仓库证据。
- 冲刺任务与进度：[`06-plan#冲刺关键路径`](../06-plan.md#冲刺关键路径进度-ssot)、[`10-progress`](../10-progress.md)。
- 源题 PDF **§7.1（官方资源池）整节忽略**，不作为设计或验收义务；§7.1 中的示例凭据不转写进派生文档，实现仓库只能通过环境变量或部署密钥引用凭据。

## 结论

文档与实现分离不是降低目标，而是保证设计目标清晰：本文档仓库定义 Conduit 超级个体的端到端交付蓝图，实现仓库负责完成真实演示闭环。
